• Java
    • We get a set of input URLs which act as a root or a starting point for our crawler.
    • The links from these initial URLs are extracted using a library called jsoup which is a HTML parser for java.
    • These "crawled" URLs are added to a queue and each of these URLs again act as a starting point for a crawler and the processes continues.
    • Since the Java crawler is multi-threaded, it can crawl at speeds of 10-30 URLs per sec.
    • These URLs along with their title andother infomation are added to a MySQL database as they are crawled because we need this data to build our search engine.

  • JSP-Servlet
    • The frontend, i.e the JSPs and servlets retrieve data from the database of crawled URLs by getting keyword(s) from the user as input.
    • The servlet searches the database for relevant URLs and sends it to the JSP to display it on the webpage.
